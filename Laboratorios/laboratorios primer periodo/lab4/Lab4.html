Respuestas a Preguntas HTML y HTTP

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet: Es una red global de computadoras interconectadas. Es la infraestructura que permite la comunicación.

World Wide Web: Es un sistema de información que utiliza Internet para acceder a documentos y recursos interconectados a través de URLs.

2. ¿Cuáles son las partes de un URL?

  • Protocolo: Ej. HTTP o HTTPS.
  • Dominio: Nombre del sitio web (ej. www.ejemplo.com).
  • Puerto: Opcional, especifica un puerto específico (ej. :80, :443).
  • Ruta: Ubicación del recurso dentro del sitio.
  • Parámetros: Información adicional (ej. ?id=123).
  • Fragmento: Ancla o referencia interna (ej. #seccion).

3. ¿Cuál es el propósito de los métodos HTTP?

  • GET: Recuperar datos.
  • HEAD: Recuperar cabeceras de respuesta.
  • POST: Enviar datos para ser procesados.
  • PUT: Actualizar o reemplazar un recurso.
  • PATCH: Modificar parcialmente un recurso.
  • DELETE: Eliminar un recurso.

4. Método HTTP para enviar un formulario HTML

POST: Se utiliza para enviar formularios, como usuario y contraseña, porque es más seguro al no incluir datos en la URL.

5. Método HTTP para acceder a una página web

GET: Es el método utilizado para recuperar recursos como páginas web.

6. Respuesta HTTP con código 200

El código 200 significa éxito. No ocurrió ningún error.

7. ¿Responsabilidad del error 404?

No siempre es responsabilidad del desarrollador, ya que puede deberse a un enlace incorrecto creado por el usuario. Sin embargo, el desarrollador debería manejar estos errores adecuadamente con mensajes claros.

8. ¿Responsabilidad del error 500?

Sí, normalmente es responsabilidad del desarrollador, ya que indica un problema interno en el servidor que necesita ser corregido.

9. Atributo depreciado en HTML5

Un atributo depreciado significa que no se recomienda usarlo, ya que podría ser eliminado en el futuro. Ejemplos:

  • <font> (HTML 4, reemplazado por CSS).
  • <center> (HTML 4, reemplazado por CSS).

10. Diferencias principales entre HTML 4 y HTML5

  • Soporte para elementos multimedia (audio y video).
  • Mejoras en formularios (nuevos controles como color, fecha, etc.).
  • Uso de etiquetas semánticas como <article> y <section>.

11. Componentes de una tabla

  • Estructura: <table>, <tr>, <th>, <td>.
  • Estilo: Bordes, fondos, espaciado, etc.

12. Principales controles de una forma HTML5

  • Entrada de color, fecha, email, número, rango, búsqueda, tel, URL.

13. Soporte HTML5 en tu navegador

Puedes verificarlo en HTML5 Test. Según tu sistema (Windows 11), probablemente uses un navegador moderno como Microsoft Edge o Google Chrome, los cuales tienen un soporte muy alto para HTML5.

Preguntas Laboratorio 3

14. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

  • Usarse para evitar reglas heredadas, podemos usar !important para dar prioridad a algún elemento.
  • Si trabajamod con estilos globales y necesitamos asegurarnos de que una regla no sea modificada.
  • Hay que limitar su uso pues, puede causar problemas como mantenimiento difícil o conflictos en estilos dinámicos.

15. ¿Por qué debe escogerse con cuidado una imagen de fondo en HTML?

Seleccionar una imagen de fondo debe hacerse con precaución porque puede afectar la legibilidad del contenido, el rendimiento de la página y la experiencia del usuario.

  • Legibilidad: Si la imagen es demasiado detallada o tiene colores similares al texto, puede dificultar la lectura.
  • Rendimiento: Imágenes de alta resolución pueden ralentizar el tiempo de carga, especialmente en dispositivos móviles.
  • Compatibilidad: Algunas imágenes pueden no verse bien en diferentes tamaños de pantalla, por lo que es recomendable usar estilos como `background-size: cover;` para ajustarlas adecuadamente.

16. ¿Cómo elegir entre %, px y pt en CSS?

La elección de unidades depende del contexto y el diseño responsivo:

  • % (porcentaje): Útil para diseños flexibles y adaptables, ya que se basa en el tamaño del elemento padre.
  • px (píxeles): Ideal cuando se necesita un tamaño fijo, pero puede ser menos efectivo en diseños responsivos.
  • pt (puntos): Más común en documentos impresos, pero menos utilizado en web porque la conversión entre pantallas puede variar.

En general, se recomienda el uso de unidades relativas como `em` o `rem` para mejor escalabilidad.

17. ¿Por qué una versión minimizada del CSS mejora el rendimiento?

El uso de CSS minimizado es importante para mejorar la velocidad de carga del sitio web.

  • Reducción de tamaño: Se eliminan espacios, comentarios y caracteres innecesarios, reduciendo el peso del archivo.
  • Menos solicitudes al servidor: Si se combinan y minimizan varios archivos CSS, se reducen las peticiones HTTP.
  • Optimización en el procesamiento: Los navegadores interpretan y aplican los estilos más rápidamente.

Se recomienda el uso de herramientas como CSS Minifier o configuraciones en Webpack para automatizar este proceso.
