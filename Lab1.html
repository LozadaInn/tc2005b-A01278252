Respuestas a Preguntas HTML y HTTP

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet: Es una red global de computadoras interconectadas. Es la infraestructura que permite la comunicación.

World Wide Web: Es un sistema de información que utiliza Internet para acceder a documentos y recursos interconectados a través de URLs.

2. ¿Cuáles son las partes de un URL?

  • Protocolo: Ej. HTTP o HTTPS.
  • Dominio: Nombre del sitio web (ej. www.ejemplo.com).
  • Puerto: Opcional, especifica un puerto específico (ej. :80, :443).
  • Ruta: Ubicación del recurso dentro del sitio.
  • Parámetros: Información adicional (ej. ?id=123).
  • Fragmento: Ancla o referencia interna (ej. #seccion).

3. ¿Cuál es el propósito de los métodos HTTP?

  • GET: Recuperar datos.
  • HEAD: Recuperar cabeceras de respuesta.
  • POST: Enviar datos para ser procesados.
  • PUT: Actualizar o reemplazar un recurso.
  • PATCH: Modificar parcialmente un recurso.
  • DELETE: Eliminar un recurso.

4. Método HTTP para enviar un formulario HTML

POST: Se utiliza para enviar formularios, como usuario y contraseña, porque es más seguro al no incluir datos en la URL.

5. Método HTTP para acceder a una página web

GET: Es el método utilizado para recuperar recursos como páginas web.

6. Respuesta HTTP con código 200

El código 200 significa éxito. No ocurrió ningún error.

7. ¿Responsabilidad del error 404?

No siempre es responsabilidad del desarrollador, ya que puede deberse a un enlace incorrecto creado por el usuario. Sin embargo, el desarrollador debería manejar estos errores adecuadamente con mensajes claros.

8. ¿Responsabilidad del error 500?

Sí, normalmente es responsabilidad del desarrollador, ya que indica un problema interno en el servidor que necesita ser corregido.

9. Atributo depreciado en HTML5

Un atributo depreciado significa que no se recomienda usarlo, ya que podría ser eliminado en el futuro. Ejemplos:

  • <font> (HTML 4, reemplazado por CSS).
  • <center> (HTML 4, reemplazado por CSS).

10. Diferencias principales entre HTML 4 y HTML5

  • Soporte para elementos multimedia (audio y video).
  • Mejoras en formularios (nuevos controles como color, fecha, etc.).
  • Uso de etiquetas semánticas como <article> y <section>.

11. Componentes de una tabla

  • Estructura: <table>, <tr>, <th>, <td>.
  • Estilo: Bordes, fondos, espaciado, etc.

12. Principales controles de una forma HTML5

  • Entrada de color, fecha, email, número, rango, búsqueda, tel, URL.
